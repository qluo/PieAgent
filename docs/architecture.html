Pi Agent Architecture

One agent orchestrates tools. A separate face controller thread renders the current visual state. The only shared contract is FaceState.

1. Agent Design

Agent

Owns the assistant loop and decides what happens next.

  • Waits for wake word
  • Runs speech-to-text
  • Calls LLM and tools
  • Runs text-to-speech
  • Updates FaceState

2. Face Controller Design

Agent Thread

Writes simple semantic states.

face_state.set("listening") face_state.set("thinking") face_state.set("speaking") face_state.set("idle")

Shared FaceState

Thread-safe shared state protected by a lock.

state = face_state.get()

Face Controller Thread

Continuously reads FaceState and asks the renderer to draw the matching animation.

  • Loads face assets
  • Chooses animation frames
  • Draws to Pi display

3. Agentic Loop

Idle

Face shows idle while the wake word tool listens.

Wake

Wake word detected. Agent moves into active interaction.

Listen

Face shows listening. STT captures the user request.

Think

Face shows thinking. A local Ollama LLM may call tools such as search.

Speak

Face shows speaking. TTS plays the answer.

Return

Agent returns to idle and waits for the next wake word.
