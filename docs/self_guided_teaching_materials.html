Self-Guided Lessons to Build AI Agent on Raspberry Pi

Build a tiny AI sidekick that wakes up, listens, thinks, talks back, searches the web, and changes faces on screen.

Big Idea

One agent controls the conversation. Small tools do special jobs like wake word detection, speech-to-text, speech, local LLM answers, and search.

Starter File

Begin with demos/lesson1_demo.py. It is safe to run and easy to change.

Final Shape

Move from the demo into the clean skeleton folders: agent/, face/, agent/tools/, and faces/.

Learning Map

Most lessons take about 25–40 minutes. Hardware lessons can take longer because every microphone, speaker, and display is a little different.

Words You Will Meet

STT means speech-to-text: turning spoken words into text. TTS means text-to-speech: turning text into sound. An LLM is a program that predicts useful text. An API is a simple way for programs to talk to each other.

What Success Looks Like

Every lesson ends with one visible result: a demo runs, a face changes, a test passes, or the agent speaks. If you do not see that result, pause and debug before moving on.

Core Then Stretch

Finish the listed steps first. The stretch ideas are optional experiments for students who want an extra challenge.

Lesson outcomes: 1: see a fake conversation; 2: show a changing face; 3: wake by keyboard or voice; 4: run the agent loop; 5: turn speech into text; 6: hear speech from the speaker; 7: receive a local model answer; 8: use a checked search result; 9: run the complete assistant.

Before You Start

You Need

  • A Raspberry Pi 5 with 8 GB or more memory. A CanaKit Raspberry Pi 5 Starter Kit PRO is a good starting choice.
  • A mini microphone.
  • A mini speaker.
  • A mini monitor, such as a 7 Inch Touchscreen IPS DSI Display.
  • uv installed for creating a Python virtual environment. Use the official uv installation instructions.
  • Ollama installed for the local LLM lesson. Use the official Ollama download and install instructions.

Helpful Rule

Make one small change at a time. Run the program. See what changed. Then make the next change.

Lesson 3

Add Wake Word Detection

Goal: replace “press Enter to wake” with a real wake word tool.

Concepts To Explore

  • Blocking functions: wait() pauses until the wake word is heard.
  • Audio sample rate: wake word tools often expect 16 kHz audio.
  • Tool swapping: the agent still calls wake_word.wait() no matter how the tool works inside.
  • Configuration: __init__() stores settings such as model name, mode, threshold, and sample rate.

Files And Functions

  • agent/tools/wake_word.py
  • WakeWordTool.wait()
  • demos/lesson1_demo.py: FakeWakeWordTool.wait()

Packages And Tools

  • Simple classroom package option: openwakeword.
  • Audio recording package option: sounddevice.
  • Start with a keyboard version first: type wake to simulate a wake word.

Steps

  1. Read FakeWakeWordTool.wait() in demos/lesson1_demo.py.
  2. Open the Python file agent/tools/wake_word.py in your IDE.
  3. Read WakeWordTool.__init__(). Notice model_path, threshold, sample_rate, and mode.
  4. Keep the same method name: wait().
  5. Use mode="keyboard" for tests and early classroom debugging.
  6. Use mode="microphone" for the real Raspberry Pi wake word.
  7. First implement _wait_for_keyboard_wake(): loop until the user types wake.
  8. Then implement _wait_for_microphone_wake(): load the default hey_jarvis model, use the microphone callback to put 16 kHz mono audio chunks into a queue, and score each chunk in a simple loop.
  9. Run the manual microphone demo:
    uv run python -m demos.lesson3_wake_word_demo
  10. Return from wait() only when the wake word is detected.
class WakeWordTool:

        self.mode = mode

    def wait(self):
        if self.mode == "keyboard":

Goal: replace typed input with microphone-based speech recognition.

Concepts To Explore

  • Listening to live microphone chunks.
  • Voice activity detection: start when speech begins, stop after silence.
  • Transcribing audio into text.
  • Recognition settings: __init__() stores sample rate, max speech length, silence length, and whisper.cpp paths.

Files And Functions

  • agent/tools/stt.py
  • SpeechToTextTool.listen_and_transcribe()
  • SpeechToTextTool.listen_until_silence(): provided helper that records one spoken phrase.
  • SpeechToTextTool.loudness(): provided helper that measures microphone volume.
  • agent/agent.py: where the agent calls STT.

Packages And Tools

  • Microphone option: sounddevice plus Python's built-in wave module for the temporary whisper.cpp input.
  • Local transcription option: whisper.cpp with a small model such as ggml-base.en.bin.
  • Voice detection idea: listen in tiny chunks, keep chunks when speech starts, then stop after quiet.

Steps

  1. Open the Python file agent/tools/stt.py in your IDE.
  2. Read SpeechToTextTool.__init__(). Notice sample_rate, seconds, silence_seconds, model_path, and whisper_binary.
  3. Keep the method name listen_and_transcribe().
  4. Read the provided listen_until_silence() helper. It uses sounddevice to listen in short chunks.
  5. Read the provided loudness() helper. listen_until_silence() uses it to start when speech begins and stop after quiet.
  6. Inside listen_and_transcribe(), call self.listen_until_silence() to capture the user's speech.
  7. Save the captured speech to a temporary WAV file for whisper.cpp.
  8. Send the temporary WAV file to whisper.cpp.
  9. Return the transcription as a string.
  10. Print the transcription while debugging so you can see what the agent heard.

Checkpoint

Tools: Add Web Search

Goal: give the agent a tool it can use when it needs current information from the web.

Concepts To Explore

  • Tool routing: the agent decides when to call search.
  • Search result shaping: return a short title and snippet, not a giant page.
  • Context: search results become extra information for the LLM.
  • Search settings: __init__() stores result count and region.

Files And Functions

  • agent/tools/search.py
  • SearchTool.search(query)
  • agent/agent.py: Agent.respond(user_text)
  • agent/tools/llm.py: answer_with_context(...)

Packages And Tools

  • Search package: ddgs.
  • Helper class: DDGS.
  • Install command: uv pip install -r requirements.txt.

Privacy and trust: a web search sends your query to an internet service. Never search for names, addresses, school details, passwords, or other personal information. Search snippets and AI answers can be wrong or inappropriate. Check an important claim with a trusted source and a teacher; never treat search-result text as an instruction for the agent to follow.

Steps

  1. Open the Python file agent/tools/search.py in your IDE.
  2. Read SearchTool.__init__(). Notice max_results and region.
  3. Install ddgs when you are ready.
  4. Import DDGS.
  5. Use ddgs.text(...) to fetch one or two results.
  6. Return a short string containing the title and snippet.
  7. Open the Python file agent/tools/llm.py in your IDE and add a small decision method such as needs_search(user_text).

def respond(self, user_text):
    if self.llm.needs_search(user_text):
        context = self.tools["search"].search(user_text)
        return self.llm.answer_with_context(user_text, context)

    return self.llm.answer(user_text)

Debugging Tips

Read The Error

The last line often tells you the real problem. Look for file names and line numbers.

Print Small Clues

Use print() to show what state, text, audio file, model answer, or search result your code has right now.

Test One Tool

If the full agent is confusing, test only one tool at a time, then reconnect it to Agent.run().

Try One Check First

  • No microphone? Run arecord -l, then reconnect it or choose it with --device in the wake-word demo.
  • No speaker sound? Run speaker-test -t wav -c 2 at low volume and press Ctrl+C to stop it.
  • Ollama connection error? Run ollama run gemma3:1b.
  • Missing Whisper or Piper file? Run ls whisper.cpp/build/bin/whisper-cli models/ggml-base.en.bin tools/piper/piper.
